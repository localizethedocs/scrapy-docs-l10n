扩展

Extensions are components that allow inserting your own custom functionality into Scrapy.

与其他组件不同，扩展在废料中没有特定的角色。 它们是“通配符”组件，可用于任何不适合任何其他类型组件角色的组件。

正在加载和激活扩展

每次运行时创建一个单一的扩展类实例，在启动时加载扩展。

To enable an extension, add it to the EXTENSIONS setting. For example:

EXTENSIS = Power
    "scrapy.extensions.corests.CoreStats": 500,
    "scrapy.extensions.telnet.TelnetConsole": 500,
}

EXTENSIONS is merged with EXTENSIONS_BASE (not meant to be overridden), and the priorities in the resulting value determine the loading order.

As extensions typically do not depend on each other, their loading order is irrelevant in most cases. This is why the EXTENSIONS_BASE setting defines all extensions with the same order (0). However, you may need to carefully use priorities if you add an extension that depends on other extensions being already loaded.

写入您自己的扩展

Each extension is a component.

Typically, extensions connect to signals and perform tasks triggered by them.

示例扩展

我们将在这里执行一个简单的扩展来说明上一节描述的概念。这个扩展将每次记录一条消息：

  • 一个蜘蛛已经打开

  • 一个蜘蛛已被关闭

  • 特定数量的物品已被擦除

扩展将通过 MYEXT_ENABLED 设置启用，项目数量将通过 MYEXT_ITEMCOUNT 设置。

这是此扩展的代码：

import logging
from scrapy import signals
from scrapy.exceptions import NotConfigured

logger = logging.getLogger(__name__)


class SpiderOpenCloseLogging:
    def __init__(self, item_count):
        self.item_count = item_count
        self.items_scraped = 0

    @classmethod
    def from_crawler(cls, crawler):
        # first check if the extension should be enabled and raise
        # NotConfigured otherwise
        if not crawler.settings.getbool("MYEXT_ENABLED"):
            raise NotConfigured

        # get the number of items from settings
        item_count = crawler.settings.getint("MYEXT_ITEMCOUNT", 1000)

        # instantiate the extension object
        ext = cls(item_count)

        # connect the extension object to signals
        crawler.signals.connect(ext.spider_opened, signal=signals.spider_opened)
        crawler.signals.connect(ext.spider_closed, signal=signals.spider_closed)
        crawler.signals.connect(ext.item_scraped, signal=signals.item_scraped)

        # return the extension object
        return ext

    def spider_opened(self, spider):
        logger.info("opened spider %s", spider.name)

    def spider_closed(self, spider):
        logger.info("closed spider %s", spider.name)

    def item_scraped(self, item, spider):
        self.items_scraped += 1
        if self.items_scraped % self.item_count == 0:
            logger.info("scraped %d items", self.items_scraped)

内置扩展引用

一般用途扩展

日志统计扩展

class scrapy.extensions.logstats.LogStats[源代码]

记录基本统计信息，如破解过的页面和被窃的项目。

核心统计扩展

class scrapy.extensions.corestats.CoreStats[源代码]

Enable the collection of core statistics, provided the stats collection is enabled (see Stats Collection).

日志计数扩展

class scrapy.extensions.logcount.LogCount(crawler: Crawler)[源代码]

安装日志处理程序，按级别记录日志消息。

The handler installed is scrapy.utils.log.LogCounterHandler. The counts are stored in stats as log_count/<level>.

在 2.14 版本加入.

Telnet 控制台扩展

class scrapy.extensions.telnet.TelnetConsole[源代码]

提供一个 telnet 控制台，用于进入当前运行的 Scrapy 进程中的 Python 解释器，这对调试非常有用。

The telnet console must be enabled by the TELNETCONSOLE_ENABLED setting, and the server will listen in the port specified in TELNETCONSOLE_PORT.

内存使用扩展

class scrapy.extensions.memusage.MemoryUsage[源代码]

备注

此扩展在 Windows 中不可用。

监视运行旋转器的废料过程所使用的内存，并且：

  1. sends a memusage_warning_reached signal when it exceeds MEMUSAGE_WARNING_MB

  2. closes the spider with the "memusage_exceeded" reason when it exceeds MEMUSAGE_LIMIT_MB

This extension is enabled by the MEMUSAGE_ENABLED setting and can be configured with the following settings:

  • MEMUSAGE_LIMIT_MB

  • MEMUSAGE_WARNING_MB

  • MEMUSAGE_CHECK_INTERVAL_SECONDS

内存调试器扩展

class scrapy.extensions.memdebug.MemoryDebugger[源代码]

调试内存使用的扩展。它收集有关以下方面的信息：

  • Python垃圾收集器未收集的对象

  • objects left alive that shouldn't. For more info, see Debugging memory leaks with trackref

To enable this extension, turn on the MEMDEBUG_ENABLED setting. The info will be stored in the stats.

Spider 状态扩展

class scrapy.extensions.spiderstate.SpiderState[源代码]

将其加载到一个龙格之前并保存到后面来管理旋转器状态数据。

Give a value to the JOBDIR setting to enable this extension. When enabled, this extension manages the state attribute of your Spider instance:

  • When your spider closes (spider_closed), the contents of its state attribute are serialized into a file named spider.state in the JOBDIR folder.

  • When your spider opens (spider_opened), if a previously-generated spider.state file exists in the JOBDIR folder, it is loaded into the state attribute.

For an example, see Keeping persistent state between batches.

关闭蜘蛛扩展

class scrapy.extensions.closespider.CloseSpider[源代码]

在满足某些条件时自动关闭旋转器，使用每个条件的特定关闭原因。

关闭旋转器的条件可以通过以下设置进行配置：

  • CLOSESPIDER_TIMEOUT

  • CLOSESPIDER_TIMEOUT_NO_ITEM

  • CLOSESPIDER_ITEMCOUNT

  • CLOSESPIDER_PAGECOUNT

  • CLOSESPIDER_ERRORCOUNT

备注

When a certain closing condition is met, requests which are currently in the downloader queue (up to CONCURRENT_REQUESTS requests) are still processed.

PLAYLIST_TIMEOUT

默认值: 0

An integer which specifies a number of seconds. If the spider remains open for more than that number of seconds, it will be automatically closed with the reason closespider_timeout. If zero (or non set), spiders won't be closed by timeout.

PLAYLIST_DESC

默认值: 0

指定数秒的整数。 如果蜘蛛在最后几秒钟内没有生产任何物品，它将被关闭，原因是 closespider_timeout_no_item。 如果零(或未设置)，则除非没有生产任何物品，否则将不会关闭。

PLAYLIST_TITLE

默认值: 0

指定项数的整数。 如果旋转器碎片超过该数量，并且这些物品由项目管道传输， 蜘蛛将被关闭，原因是 closespider_itemcount 。 如果为零(或未设置)，拼写器将不会被通过项目的数量关闭。

PLAYLIST_DESCRIPTION

默认值: 0

一个整数型变量，它指定的 crawl 响应的最大数量。 如果超越那个范围，那个蜘蛛将被关闭，原因是 closespider_pagecount 。 如果零(或未设置)，喷雾器将不会因为被压缩的响应数而关闭。

PLAYLIST_TITLE

默认值: 0

一个整数型变量，它指定了无项目拆卸的连续响应的最大数。 如果旋转器弹头的连续响应超过连续响应，同时没有物品被打破。 蜘蛛将被关闭，原因是 closespider_pagecount_no_item 。 如果零(或未设置)，喷雾器将不会被无项目的被拖动响应数所关闭。

CLESSPIDER_ERRORCOUNT

默认值: 0

在关闭拼写器之前指定接收的最大错误数的整数。 如果蜘蛛生成的错误超过了这个数量，它将被关闭，原因是 closespider_errorcount 。 如果零(或未设定)，喷洒器将不会因数个错误而关闭。

此扩展记录以下可配置部分：

  • "delta" 显示自上次统计日志消息以来一些数字统计有多大变化。

    The PERIODIC_LOG_DELTA setting determines the target stats. They must have int or float values.

  • "stats" 显示了某些统计数据的当前值。

    The PERIODIC_LOG_STATS setting determines the target stats.

  • "time" 会显示详细的计时数据。

    The PERIODIC_LOG_TIMING_ENABLED setting determines whether or not to show this section.

This extension logs data at the start, then on a fixed time interval configurable through the LOGSTATS_INTERVAL setting, and finally right before the crawl ends.

示例扩展配置：

custom_settings = {
    "LOG_LEVEL": "INFO",
    "PERIODIC_LOG_STATS": {
        "include": ["downloader/", "scheduler/", "log_count/", "item_scraped_count/"],
    },
    "PERIODIC_LOG_DELTA": {"include": ["downloader/"]},
    "PERIODIC_LOG_TIMING_ENABLED": True,
    "EXTENSIONS": {
        "scrapy.extensions.periodic_log.PeriodicLog": 0,
    },
}
PERIODIIC_LOG_DELTA

默认值： None

  • "PERIODIIC_LOG_DELTA": True - 显示所有的 int 和 float 统计数值。

  • "PERIODIIC_LOG_DELTA": {"include": ["downloader", "scheduler/"]} - 显示含有任何已配置子字符串的数据的三角洲。

  • "PERIODIIC_LOG_DELTA": {"exclude": ["downloader/"]} - 为所有不含任何配置的子字符串的状态显示三角洲。

PLAYLIST_DESCRIPTION

默认值： None

  • "PERIODIIC_LOG_STATS": True - 显示所有统计状态的当前值。

  • "PERIODIIC_LOG_STATS": {"include": ["downloader/", "scheduler/"]} - 显示包含任何配置的子字符串的当前统计信息。

  • "PERIODIIC_LOG_STATS": {"exclude ": ["downloader/"]} - 显示所有不含任何已配置子字符串的统计数据的当前值。

PERIODIIC_LOG_TIMING_DESCRIPTION

默认值： False

True 能够记录计时数据(例如``"time"`` 部分)。

调试扩展

堆栈跟踪转储扩展

class scrapy.extensions.periodic_log.StackTraceDump

当收到`SIGQUIT`_ 或 `SIGUSR2 信号时有关运行过程的信息。丢失的信息如下：

  1. 引擎状态 (使用 scrapy.utils.engine.get_engine_status() )

  2. live references (see Debugging memory leaks with trackref)

  3. 所有线程的堆栈跟踪

堆栈跟踪和引擎状态被倾弃后，废料处理继续正常运行。

这个扩展仅适用于与 POSIX兼容的平台(即不适用于Windows)，因为“SIGQUIT`_ 和 “SIGUSR2 信号”在Windows上不可用。

至少有两种方式发送废件`SIGQUIT`_ 信号：

  1. 当Scrapy process 正在运行时按 Ctrl (仅限Linux？)

  2. 通过运行此命令(假定``<pid>`` 是Scrapy进程的进程ID):

    击杀-QUIT <pid>

调试器扩展

class scrapy.extensions.periodic_log.Debugger

Invokes a Python debugger inside a running Scrapy process when a SIGUSR2 signal is received. After the debugger is exited, the Scrapy process continues running normally.

此扩展仅适用于 POSIX兼容平台(即不适用于 Windows)。
