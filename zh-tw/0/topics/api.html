核心 API

在 0.15 版本新加入.

本节记载了Scrapy core API，它的对象是扩展和中间件的开发者。

Crawler API

The main entry point to Scrapy API is the Crawler object, passed to extensions through the from_crawler class method. This object provides access to all Scrapy core components, and it’s the only way for extensions to access them and hook their functionality into Scrapy.

The Extension Manager is responsible for loading and keeping track of installed extensions and it’s configured through the EXTENSIONS setting which contains a dictionary of all available extensions and their order similar to how you configure the downloader middlewares.

class scrapy.crawler.Crawler(settings)

The Crawler object must be instantiated with a scrapy.settings.Settings object.

settings

此Crawler的设置管理器。

此选项被扩展 & 中间器用于访问此 crawler 的Scrapy 设置。

For an introduction on Scrapy settings see 設定.

For the API see Settings class.

signals

这个爬行器的信号管理器。

扩展和中间件用于将自己绑定到废料功能。

For an introduction on signals see Signals.

For the API see SignalManager class.

stats

这个crawler的数据收集器。

这是从扩展和中期来记录他们的行为统计或其他扩展收集的访问统计信息。

For an introduction on stats collection see Stats Collection.

For the API see StatsCollector class.

extensions

用于跟踪启用扩展的扩展管理器。

大多数扩展不需要访问此属性。

For an introduction on extensions and a list of available extensions on Scrapy see 扩展.

spiders

The spider manager which takes care of loading and instantiating spiders.

大多数扩展不需要访问此属性。

engine

执行引擎，它协调调度器、下载器和蜘蛛之间的核心爬行逻辑。

Some extension may want to access the Scrapy engine, to modify inspect or modify the downloader and scheduler behaviour, although this is an advanced use and this API is not yet stable.

configure()

Configure the crawler.

This loads extensions, middlewares and spiders, leaving the crawler ready to be started. It also configures the execution engine.

start()

Start the crawler. This calls configure() if it hasn’t been called yet. Returns a deferred that is fired when the crawl is finished.

設定 API

scrapy.settings.SETTINGS_PRIORITIES

设置在废件中默认设置优先级的关键名称和优先级的字典。

Each item defines a settings entry point, giving it a code name for identification and an integer priority. Greater priorities take more precedence over lesser ones when setting and retrieving values in the Settings class.

SETTINGS_PRIORITIES = {
    'default': 0,
    'command': 10,
    'project': 20,
    'cmdline': 40,
}

For a detailed explanation on each settings sources, see: 設定.

class scrapy.settings.Settings(values={}, priority='project')

此对象储存内部组件配置的 Scrapy 设置，可以用于任何进一步的定制操作。

After instantiation of this class, the new object will have the global default settings described on Built-in settings reference already populated.

Additional values can be passed on initialization with the values argument, and they would take the priority level. If the latter argument is a string, the priority name will be looked up in SETTINGS_PRIORITIES. Otherwise, a expecific integer should be provided.

Once the object is created, new settings can be loaded or updated with the set() method, and can be accessed with the square bracket notation of dictionaries, or with the get() method of the instance and its value conversion variants. When requesting a stored key, the value with the highest priority will be retrieved.

set(name, value, priority='project')

保存一个具有给定优先级的密钥/值属性。

Settings should be populated before configuring the Crawler object (through the configure() method), otherwise they won’t have any effect.

參數

  • name (string) – 设置名称

  • value (any) – 要与设置关联的值

  • priority (string or int) – the priority of the setting. Should be a key of SETTINGS_PRIORITIES or an integer

setdict(values, priority='project')

存储具有特定优先级的密钥/值对。

This is a helper function that calls set() for every item of values with the provided priority.

參數

  • values (dict) – 设置名称和值

  • priority (string or int) – the priority of the settings. Should be a key of SETTINGS_PRIORITIES or an integer

setmodule(module, priority='project')

从具有特定优先级的模块存储设置。

This is a helper function that calls set() for every globally declared uppercase variable of module with the provided priority.

參數

  • module (module object or string) – 模块或模块路径

  • priority (string or int) – the priority of the settings. Should be a key of SETTINGS_PRIORITIES or an integer

get(name, default=None)

获取一个设置值而不影响其原始类型。

參數

  • name (string) – 设置名称

  • default (any) – 如果没有找到设置则返回的值

getbool(name, default=False)

Get a setting value as a boolean. For example, both 1 and '1', and True return True, while 0, '0', False and None return False``

例如，环境变量设置为 '0' 时返回的 False 。

參數

  • name (string) – 设置名称

  • default (any) – 如果没有找到设置则返回的值

getint(name, default=0)

Get a setting value as an int

參數

  • name (string) – 设置名称

  • default (any) – 如果没有找到设置则返回的值

getfloat(name, default=0.0)

Get a setting value as a float

參數

  • name (string) – 设置名称

  • default (any) – 如果没有找到设置则返回的值

getlist(name, default=None)

Get a setting value as a list. If the setting original type is a list it will be returned verbatim. If it’s a string it will be split by 「,」.

例如，通过环境变量填充的设置设置设置为 'one,two' 时，会返回一个列表[『one』,』2』]。

參數

  • name (string) – 设置名称

  • default (any) – 如果没有找到设置则返回的值

信号 API

class scrapy.signalmanager.SignalManager
connect(receiver, signal)

将接收器函数与信号连接起来。

The signal can be any object, although Scrapy comes with some predefined signals that are documented in the Signals section.

參數

  • receiver (callable) – 要连接的函数

  • signal (object) – 要连接到的信号

send_catch_log(signal, \*\*kwargs)

发送一个信号，捕获异常并记录它们。

The keyword arguments are passed to the signal handlers (connected through the connect() method).

send_catch_log_deferred(signal, \*\*kwargs)

Like send_catch_log() but supports returning deferreds from signal handlers.

Returns a deferred that gets fired once all signal handlers deferreds were fired. Send a signal, catch exceptions and log them.

The keyword arguments are passed to the signal handlers (connected through the connect() method).

disconnect(receiver, signal)

Disconnect a receiver function from a signal. This has the opposite effect of the connect() method, and the arguments are the same.

disconnect_all(signal)

断开所有接收器与给定信号的连接。

參數

signal (object) – 要断开连接的信号

统计数据收集器 API

There are several Stats Collectors available under the scrapy.statscol module and they all implement the Stats Collector API defined by the StatsCollector class (which they all inherit from).

class scrapy.statscol.StatsCollector
get_value(key, default=None)

返回给定状态键值或默认值，如果它不存在。

get_stats()

从当前运行的蜘蛛获取所有的统计信息作为一种词典。

set_value(key, value)

设置给定统计键的给定值。

set_stats(stats)

使用 stats 参数覆盖当前的统计信息。

inc_value(key, count=1, start=0)

通过给定的计数增加给定统计数据的值，假定给定的起始值 (当它没有设置时)。

max_value(key, value)

仅在同一键的当前值小于值时设置给定的键值。 如果给定的键值没有当前值，则该值总是被设置的。

min_value(key, value)

只有当同一键的当前值大于值时，才设置给定的键值。 如果给定的键值没有当前值，则该值总是被设置的。

clear_stats()

清除所有数据

以下方法不是统计数据收集api的一部分，而是在实现自定义统计数据收集器时使用：

open_spider(spider)

Open the given spider for stats collection.

close_spider(spider)

Close the given spider. After this is called, no more specific stats can be accessed or collected.
